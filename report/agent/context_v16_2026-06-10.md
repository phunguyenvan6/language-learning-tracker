## Key Mental Model Mappings (React Native -> SwiftUI) [[CHANGED]]

| React Native / TypeScript | Swift / SwiftUI |
|---|---|
| `const` | `let` |
| `let` | `var` |
| `interface` / object type | `struct` |
| Redux/Zustand store | `@Observable` class (Observation) |
| `useState(store)` + mutate | `@State` owner + `@Observable` property tracking |
| Controlled component | `@Binding` — parent `$field`, child `@Binding var field` |
| `value` + `onChangeText` | `TextField(..., text: $field)` |
| `parseFloat` / `isNaN` | `Double(string)` → `Double?` |
| Form `error` + disabled submit | `errorMessage` + `isFormValid` computed |
| Context | `@Environment` (Day 3) |
| `useEffect(() => {}, [])` | `.task` / `.onAppear` (Day 3) |
| `useEffect(..., [value])` | `.onChange(of:)` — da dung cho success banner |
| Callback prop | Closure `(String) -> Void` |
| Mock store Storybook | `init(manager:)` + `previewSeeded()` |
| Preview local state | `@Previewable @State` trong `#Preview` |
| Theme tokens | `AppColor` / `AppSpacing` + `formField()` modifier |

## Common Mistakes to Watch For [[CHANGED]]

1. Side effects trong `body` — dung `submit()`, `.onChange`, `.task`
2. ~~listRevision workaround~~ — khong can sau @Observable
3. Amount as `Double` trong TextField — giu String, parse khi validate
4. `@State` duplicate o child cho cung field — child dung `@Binding`
5. `createdAt: Date()` thay vi parameter — da fix Day 2
6. `isFormValid` chi check `!= nil` ma quen `> 0`
7. Preview `@Binding` khong co parent state — dung `@Previewable @State`
8. Force unwrap `!` trong production
