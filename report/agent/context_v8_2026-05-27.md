## Key Mental Model Mappings (React Native → SwiftUI) [[CHANGED]]

| React Native / TypeScript | Swift / SwiftUI |
|---|---|
| `const` | `let` |
| `let` | `var` |
| `interface` / object type | `struct` |
| Service / store class | `final class` |
| In-memory CRUD store | `TransactionManager` |
| `Intl.NumberFormat` | `NumberFormatter` / `Double.asVND()` |
| List item `id` | `Identifiable` |
| `string \| undefined` | `String?` |
| Redux selector | `filter(by:)`, `sortByNewest()` |
| TypeScript `interface` | Swift `protocol` |
| `async function` | `func ... async` |
| Promise rejection | `throws` / `TransactionError` |
| **Function component** | **`struct SomeView: View`** |
| **Props** | **stored properties / init** |
| **`useState`** | **`@State`** |
| **FlatList** | **`List` / `ForEach`** |
| **Flexbox column/row** | **`VStack` / `HStack`** |
| `useEffect` mount | `.onAppear` / `.task` |
| React Navigation | `NavigationStack` |

## Common Mistakes to Watch For [[CHANGED]]

1. Đặt side effects trong `body` thay vì dùng `.task` — **ưu tiên khi vào Phase 2**
2. Cố dùng Flexbox mental model thay vì `VStack/HStack/ZStack`
3. Dùng global store quá nhiều thay vì `@State` local cho UI nhỏ
4. Map hooks 1-1 sang SwiftUI
5. Force unwrap `!` trong production code
6. Gộp nhiều test `throws` trong một `do-catch` (Playground)
7. Main Actor / `Task` isolation trong Playground
8. **`update` silent khi id không tồn tại** — cân nhắc `throws` khi gắn UI [[CHANGED]]
9. Copy toàn bộ 7 playground vào App — nên tách model/manager sang file dùng chung khi build EWalletHomeScreen [[CHANGED]]
